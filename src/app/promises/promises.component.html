Promises

To understand what a promise is, we first must understand what a proxy is. A proxy, as defined in English, is:

"The authority to respresent someone else"

In terms of JavaScript, we can apply the same description the English language has provided for us. A proxy is an object that enables you to create a representation of another object. This representation can then intercept and redefine fundamental operations for the other object. So, what does this mean for a promise? Well, a promise is a proxy for an object that is not necessarily known when the promise is created.

There are 3 "States" of promises:

  • Pending: Initial state, neither fulfilled nor rejected.
  • Fulfilled: Meaning that the operation completed successfully.
  • Rejected: Meaning that the operation failed.

The promise is the stand in for a future object, a fulfilled object, or rejected object that is caught via .catch
